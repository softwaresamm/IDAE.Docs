La versión **1.2.24.0** de SAMM API amplía los servicios de definiciones, equipos y reportes técnicos, y corrige varios problemas reportados en la creación de solicitudes, remisiones y eventos.

<!-- truncate -->

## Novedades

- Se ajustó la regla de envío masivo del utilitario de reportería para que el adjunto se incluya correctamente en el correo enviado. ([SO-371](https://softwaresamm.atlassian.net/browse/SO-371))
- Se agregó la creación de controlador y de equipos para la integración con OTT. ([SO-416](https://softwaresamm.atlassian.net/browse/SO-416))
- Se habilitó el consumo de un servicio para el envío de correos según las acciones registradas en la base de datos. ([SO-459](https://softwaresamm.atlassian.net/browse/SO-459))
- El endpoint de búsqueda de estado por tipo de documento ahora recibe y mapea dinámicamente los parámetros de la URL hacia el procedimiento correspondiente. ([SO-506](https://softwaresamm.atlassian.net/browse/SO-506))
- El servicio de definiciones ahora incluye las propiedades y secciones del servicio, ampliando la información disponible para los consumidores. ([SO-514](https://softwaresamm.atlassian.net/browse/SO-514))
- Las listas de chequeo por modelo ahora permiten configurar dependencias entre sus ítems. ([SO-566](https://softwaresamm.atlassian.net/browse/SO-566))
- Se agregó la propiedad que indica si un equipo maneja horómetro, disponible para su configuración y consulta. ([SO-575](https://softwaresamm.atlassian.net/browse/SO-575))
- Se ajustó el servicio de recursos para que la aplicación móvil pueda leerlos correctamente una vez configurados. ([SO-597](https://softwaresamm.atlassian.net/browse/SO-597))
- Se agregó una nueva sección que permite controlar la visualización de los cambios de estado en la aplicación móvil. ([SO-612](https://softwaresamm.atlassian.net/browse/SO-612))
- Se controló el manejo de valores nulos en la propiedad idmob al crear un reporte, evitando fallos en el registro. ([SO-629](https://softwaresamm.atlassian.net/browse/SO-629))
- Se evitó que el servicio intente leer atributos no configurados en el modelo al consultar un serial, previniendo errores innecesarios. ([SO-643](https://softwaresamm.atlassian.net/browse/SO-643))

## Correcciones

- Se corrigió el proceso de despliegue para que ya no elimine la carpeta ScriptsBD en sa_idae. ([SO-407](https://softwaresamm.atlassian.net/browse/SO-407))
- Se solucionó un error que impedía el cargue correcto de reportes técnicos cuando incluían un atributo de tipo firma. ([SO-478](https://softwaresamm.atlassian.net/browse/SO-478))
- Se corrigió la creación de solicitudes para que ahora sí se tenga en cuenta el estado indicado. ([SO-507](https://softwaresamm.atlassian.net/browse/SO-507))
- Se corrigió el reemplazo indebido de tildes por guiones bajos en el nombre de los archivos PDF enviados por correo. ([SO-524](https://softwaresamm.atlassian.net/browse/SO-524))
- Se ajustó la creación de remisiones desde la aplicación móvil para respetar el orden de los estados y evitar que quedaran en estado cancelada de forma incorrecta. ([SO-531](https://softwaresamm.atlassian.net/browse/SO-531))
- Se corrigió la trazabilidad de los eventos creados con horómetro, que quedaba registrada sin tipo. ([SO-576](https://softwaresamm.atlassian.net/browse/SO-576))
- Se corrigió el servicio para obtener el formulario de eventos. ([SO-603](https://softwaresamm.atlassian.net/browse/SO-603))
